Selector: `app-map`  
File: `src/app/components/map/map.ts`

The `Map` component wraps [Leaflet](https://leafletjs.com/) to render an interactive map with motel/room markers and real-time user geolocation. Leaflet is loaded asynchronously (`import('leaflet')`) so it is never included in the server-side bundle.

<Info>
The component uses `isPlatformBrowser` to guard all Leaflet calls. It will not initialize on the server during Angular SSR/prerendering.
</Info>

## Interfaces

```typescript
export interface MapPoint {
  lat: number;
  lng: number;
  name: string;
  id?: number;
}
```

`MapPoint` is the shape expected by both the `points` input and the `markerClicked` output.

## Inputs and outputs

| Binding | Type | Default | Description |
|---|---|---|---|
| `@Input() points` | `MapPoint[]` | `[]` | Array of motel/room coordinates to render as markers |
| `@Input() active` | `MapPoint \| null` | `null` | When set, the map flies to this point at zoom 17 |
| `@Output() markerClicked` | `EventEmitter<MapPoint>` | — | Emitted when the user clicks a motel/room marker |

## Initialization

`ngAfterViewInit` runs the full Leaflet setup:

```typescript
async ngAfterViewInit() {
  if (!isPlatformBrowser(this.platformId)) return;

  const leaflet = await import('leaflet');
  this.L = (leaflet as any).default ?? leaflet;

  this.map = this.L.map(this.mapContainer.nativeElement, {
    center: [4.6, -74.1], // Bogotá
    zoom: 13,
  });

  this.L.tileLayer('https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png', {
    attribution: '© OpenStreetMap contributors',
  }).addTo(this.map);

  this.markerLayer = this.L.layerGroup().addTo(this.map);

  // Custom icons
  this.userIcon = this.L.icon({
    iconUrl: 'assets/icons/leaflet/icon_person.png',
    iconSize: [25, 32],
    iconAnchor: [18, 36],
    popupAnchor: [0, -32],
  });

  this.motelIcon = this.L.icon({
    iconUrl: 'assets/icons/leaflet/icon_motel.png',
    iconSize: [27, 36],
    iconAnchor: [18, 36],
    popupAnchor: [0, -36],
  });

  setTimeout(() => this.map.invalidateSize());

  if (this.points.length) {
    this.renderMarkers();
  }

  this.initializeLocation();
}
```

The default center (`[4.6, -74.1]`) is Bogotá. If any `points` arrive before `ngAfterViewInit` completes, they are rendered immediately. Geolocation initialization runs in parallel without blocking the tile layer from loading.

## Marker rendering

Every time the `points` input changes, `renderMarkers()` clears the `markerLayer` and re-adds all points:

```typescript
private renderMarkers() {
  if (!this.markerLayer) return;

  this.markerLayer.clearLayers();

  for (const p of this.points) {
    const marker = this.L.marker([p.lat, p.lng], { icon: this.motelIcon })
      .bindPopup(p.name);

    marker.on('click', () => {
      this.markerClicked.emit(p);
    });

    marker.addTo(this.markerLayer);
  }

  this.adjustView();
}
```

After rendering, `adjustView()` calls `map.setView()` for a single point or `map.fitBounds()` with 50 px padding for multiple points.

## Active point flyTo

When the `active` input changes (e.g. the user hovers a list item), the map flies smoothly to that point:

```typescript
ngOnChanges(changes: SimpleChanges) {
  if (!this.map) return;

  if (changes['points']) {
    this.renderMarkers();
    // ...
  }

  if (changes['active'] && this.active) {
    this.map.flyTo([this.active.lat, this.active.lng], 17, {
      animate: true,
      duration: 1.2,
    });
  }
}
```

## Geolocation

Geolocation is handled by a two-part approach: a one-time initial position fetch and a continuous `watchPosition` watcher.

### getUserLocation utility

File: `src/app/components/map/geolocation.ts`

```typescript
export interface UserLocation {
  latitude: number;
  longitude: number;
  accuracy: number;
}

export function getUserLocation(): Promise<UserLocation> {
  return new Promise((resolve, reject) => {
    if (!navigator.geolocation) {
      reject(new Error('Geolocalización no soportada'));
      return;
    }

    navigator.geolocation.getCurrentPosition(
      (pos) => {
        resolve({
          latitude: pos.coords.latitude,
          longitude: pos.coords.longitude,
          accuracy: pos.coords.accuracy,
        });
      },
      (err) => reject(err),
      {
        enableHighAccuracy: true,
        timeout: 15000,
        maximumAge: 0,
      }
    );
  });
}
```

### initializeLocation flow

```typescript
private async initializeLocation() {
  if (!navigator.geolocation) return;

  // One-time initial fix
  try {
    const location = await getUserLocation();
    this.userLatLng = [location.latitude, location.longitude];
    this.placeUserMarker(this.userLatLng);

    // Fly to user only if motel points are already loaded
    if (!this.userZoomDone && this.points.length > 0) {
      setTimeout(() => {
        if (this.userLatLng && !this.userZoomDone && this.points.length > 0) {
          this.userZoomDone = true;
          this.map.flyTo(this.userLatLng, 14, { animate: true, duration: 1.8 });
        }
      }, 2000);
    }
  } catch {
    console.warn('No se pudo obtener ubicación inicial');
  }

  // Real-time tracking — moves the marker only, no zoom
  this.watchId = navigator.geolocation.watchPosition(
    (pos) => {
      const newLatLng: [number, number] = [pos.coords.latitude, pos.coords.longitude];
      this.userLatLng = newLatLng;
      this.placeUserMarker(newLatLng);
    },
    (err) => console.warn('watchPosition error:', err),
    {
      enableHighAccuracy: true,
      timeout: 15000,
      maximumAge: 5000,
    }
  );
}
```

<Note>
The initial zoom to the user's position is intentionally delayed by 2 seconds and only fires once (`userZoomDone` flag). It also only fires if at least one motel point is loaded — this prevents an empty-map zoom when the API is slow.
</Note>

The `watchPosition` watcher ID is stored in `this.watchId`. On component destroy, `ngOnDestroy` calls `navigator.geolocation.clearWatch(this.watchId)` to stop tracking.

## Template

```html
<div #mapContainer class="map-container"></div>
```

The host element fills its parent via CSS:

```css
:host {
  display: block;
  width: 100%;
  height: 100%;
}
```

The parent container must have an explicit height set for the map to render correctly.

## invalidateSize

If the map is hidden and then shown (e.g. toggled with a CSS class on mobile), call `invalidateSize()` on the component reference to force Leaflet to recalculate its dimensions:

```typescript
@ViewChild(Map) mapComponent!: Map;

showMap() {
  this.mapVisible = true;
  this.mapComponent.invalidateSize();
}
```
