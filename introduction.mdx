Ubik is a PWA built with Angular that digitizes the motel booking experience. Guests can discover nearby motels on an interactive map, check room availability, and complete secure reservations — all from a mobile-optimized interface. Motel owners get a dedicated dashboard to register their establishments, manage rooms and offers, and track incoming reservations.

## What problems Ubik solves

### For guests

Finding a motel traditionally requires calling ahead, visiting booking aggregators with incomplete information, or navigating unfamiliar areas without location context. Ubik solves this by:

- Showing motels on an interactive Leaflet map centered on the guest's current geolocation.
- Displaying real-time room availability, services, and amenities for each property.
- Letting guests complete a reservation and payment entirely in-browser without leaving the platform.

### For motel owners

Small motel operators often lack affordable, purpose-built tools for managing bookings and listing their properties online. Ubik provides:

- A self-service registration flow to add their establishment with photos (uploaded via Cloudinary), location, and description.
- Per-room management: add room types, set amenities, and upload room images.
- An owner dashboard (`/dashboard/owner`) to view all reservations made by guests.
- Offer and promotion management to attract more customers.

## Platform architecture

Ubik follows a modular Angular architecture with a clear separation between UI components, business logic services, and data models.

```
src/
├── app/
│   ├── components/       # Reusable UI components (cards, map, payment modal)
│   ├── core/
│   │   ├── guards/       # Route access control (permissionGuard)
│   │   ├── interceptors/ # HTTP interceptors
│   │   ├── models/       # TypeScript interfaces (Users, Motel, Room, Reservation)
│   │   ├── services/     # Business logic (auth, motel, room, reservation, payment)
│   │   └── utils/        # Shared validation utilities
│   ├── layout/           # Header, footer, bottom nav bar
│   └── views/            # Page-level components (home, explore, dashboard, forms)
└── environments/         # Environment configuration (API URL, Cloudinary)
```

The backend is a separate service. The frontend communicates with it exclusively through the API URL defined in `environment.ts`.

<Note>
  The backend API base URL is `https://ubik-back.duckdns.org/api`. Authentication endpoints are served from `https://ubik-back.duckdns.org/auth`.
</Note>

## User roles

Ubik supports three distinct roles, controlled by `roleId` on the `Users` model and enforced at runtime by `permissionGuard`:

| Role | `roleId` | Access |
|---|---|---|
| Customer | `3` | Browse, reserve, and pay for rooms |
| Motel Owner | — | Manage properties, rooms, reservations via owner dashboard |
| Admin | — | Full platform oversight via admin dashboard |
